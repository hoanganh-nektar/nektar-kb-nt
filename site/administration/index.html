Administration

Manage Nektar settings, users, data controls, activity tracking, and revenue signal configuration — everything your RevOps team needs to keep Nektar running cleanly.

Dashboard

The dashboard displays reports and charts on the data that Nektar has added to your CRM. All boards are refreshed every 24 hours.

Note

These metrics are not intended to be business insights; their purpose is to monitor the quality and quantity of data synced by Nektar into your CRM.

Summary

This section shows "all-time" and "last 14 days" aggregations for the three key objects Nektar captures: Contacts, Activities, and Contact Roles (links between Contacts and Opportunities). Each section shows:

  • The total volume of data created or updated
  • A breakdown into different key categories, based on the object type
  • The percentage of the total CRM data volume due to Nektar
  • An estimate of the manual data entry time saved by Nektar (1 minute per contact; 30 seconds per contact role and activity)
  • The trend in the metric, comparing the latest 14-day period with the one before

Contacts

The Contacts board provides insights into contacts created by Nektar, both all-time and within the last 14 days. Key metrics include:

  • Time saved (in hours), calculated at 60 seconds per contact
  • Left Job — number of contacts with disabled email addresses (likely due to job change)
  • Number of contacts with and without a title
  • Number of contacts with and without a phone number

Contact Roles

Provides insights into contacts associated to opportunities by Nektar, both all-time and within the last 14 days. Key metrics include:

  • All-time top 5 roles added
  • New contact roles added in the last 13 weeks
  • Top 15 opportunities where contact roles are added in the last 14 days

Emails

Insights into sales-related emails captured by Nektar from users' inboxes, both all-time and within the last 14 days. Key metrics include:

  • Breakdown by sender: Us (outbound) vs. Them (inbound)
  • Breakdown by associated Salesforce object: Account, Opportunity, Lead
  • Time saved, calculated at 30 seconds per email

Events

Insights into sales-related events captured by Nektar from users' calendars, both all-time and within the last 14 days. Key metrics include:

  • Breakdown by organizer: Us (outbound) vs. Them (inbound)
  • Breakdown by associated Salesforce object: Account, Opportunity, Lead
  • Breakdown by Nektar Status:
    • Event scheduled: event has not taken place yet
    • Event deleted: the event was deleted from user's calendar
    • Event passed (no call info): start and end time has passed; no Zoom/Teams integration or data available
    • Call did not start¹: no one joined the call
    • Call < 10 mins¹: some people joined but the call lasted less than 10 minutes
    • Completed¹: several people joined and the call lasted more than 10 minutes
    ¹ Requires Zoom or Microsoft Teams integration
  • Time saved, calculated at 30 seconds per event

Users

User management lets you add and remove users, monitor license consumption, link each user to their accounts on integrated services, and monitor and control user-level sync.

The users list can be filtered by name and email address, and sorted by name or created date. Click a user's name or row to view and update details.

Editing users

In the user editing form, you can update:

  • The name displayed in user management
  • This user's Salesforce, MS365, Google, and/or Zoom user accounts
  • This user's email addresses

Nektar uses these linked external user accounts and email addresses to determine which user participated in a particular external communication.

Connectors

You can add the email address of a user under each connector you wish Nektar to read data from.

Sync toggles

These toggles let you control what data Nektar should sync for a user under a specific connector.

Admin

This toggle lets you add and remove Nektar administrators. Nektar administrators have full access to the console — they can add or remove users, manage connectors, and start or stop sync. All admins can add or remove other admins. Admins must have a Salesforce account.

Integration icons

These icons indicate a user's status within each connector. For connectors without user-level sync (such as Salesforce), a check mark is shown if the user is linked to a user account on that service. For connectors with user-level sync (Google, Microsoft, Zoom), a sync health icon is shown for every object (emails, events). Click the icon for more information and to toggle sync on or off.

Add a user

There are two ways to add users — individually or in bulk.

Click Add New User to add a single user. In the setup form, select the user from the dropdown under each connector, enable or disable relevant syncs, and optionally grant admin privileges. Click Save when done.

To add multiple users at once, click Import Users to upload a CSV file. Download the CSV template and fill in the user name, email addresses for each connector, and admin access level. After uploading, any errors are highlighted so you can correct and re-upload. Once successful, new users appear in the Users page.

Export Users

Click Export Users to download a CSV of all users including connector information, sync preferences, and admin access levels.

Bulk actions

Select one or more users with the checkboxes at the start of each row to display bulk action buttons, such as enabling or disabling a particular sync, or deleting the selected users.

Data controls

Data controls let you restrict which of your organization's emails and events are processed by Nektar. There are three kinds of controls.

Screening rules

Nektar automatically screens out emails sent by automated systems and non-personal email addresses (such as support@… and sales@…). You may additionally block emails and events under the following categories:

  • Block until date — all events with a start date and emails sent until this date will be blocked
  • Private calendar events — when toggled on, these events will not be read by Nektar
  • Email addresses — activity will not be captured if any participant matches these addresses
  • Email domains — activity will not be captured if any participant matches these domains
  • Subject line keywords — emails or events whose subject line contains any of these words will not be captured
  • Body pattern keywords — emails whose body contains any of these words will not be captured

Matching rules: case insensitive; multi-word patterns are treated as exact phrases; special characters are matched literally; line breaks, spaces, underscores, and numbers are accepted as valid prefixes or suffixes to blocked words.

Special domains and email addresses

Nektar screens out emails or events where all participants are internal to your company, tools (e.g. notifications, conference rooms), distribution lists, or personal email addresses. You may configure additional email addresses and domains as ignored — i.e. treated as internal for activity screening purposes.

Note

If an activity passes screening, all its participants are stored and processed in Nektar — including ignored and internal ones.

Write rules

Nektar only creates records that can be linked to existing Salesforce records:

  • Contacts must link to an existing account or contact via a shared, non-public domain name. Nektar will never create contacts with public domains (such as @gmail.com).
  • Activities must link to valid contacts or leads. Contacts created from participants found in the same email or event qualify.
  • Opportunity-Contact Roles are created for all contacts that have participated in activities linked to the opportunity.
Note

Write rules are in beta; contact us to implement them.

Tracker

Found at Settings > Track activities, Tracker lets you look up the path of a specific activity through Nektar's data processing pipeline. It helps understand why Nektar might have excluded a certain activity from Salesforce.

Search for an event or email by keywords in its subject line and/or the email addresses of participants.

No results?

An activity that was read by Nektar might not appear in the tracker for two reasons:

  • It is outside the time window: the tracker only displays activities read by Nektar more than 2 hours ago but less than 30 days ago.
  • The activity was filtered out by a read rule.

Revenue signals

Revenue signals let you auto-update fields on your Salesforce. You can place a request to set up a signal for any Salesforce object — Opportunity, Contact, Event, Task, Lead, Account, or Opportunity Contact Role.

Request a new signal

When you select an object, you see the list of currently available signals with their status, description, and created date. To add more, click Request for new signals. You can choose from available templates or click Start from scratch.

Once requested, a support ticket is raised and the Nektar customer team works on your request, confirms the desired outcome, and publishes it to start auto-populating fields on your Salesforce.

Signal from template

You can request to add a template as-is or request edits to it. Editing an existing template will not change the template but creates a new signal request from it.

Signal from scratch

  1. Choose either an existing Salesforce field or create a new field.
  2. Field type will be automatically populated if you select an existing Salesforce field.
  3. Define your rule in simple sentences. Include as many details as you wish.
  4. Submit request.

Track your pending requests on the signal list view.

Bulk actions

Select one or more signals with the checkboxes at the start of each row to display bulk action buttons — publish, un-publish, or delete.
