FAQs

Answers to common questions about activities, contacts, admin controls, and security.

Activities

With which account or opportunity would an activity be linked?

Nektar uses proprietary affinity models to determine this. See Graph inference for more information.

An activity is missing in Salesforce. Why?
  1. Activities matching read screening rules are not stored or processed by Nektar, and any that match write screening rules are not written to Salesforce. Verify that no screening rules match the missing activity.
  2. Nektar only captures activities where at least one participant is an existing Salesforce lead or contact, or whose email address has the same domain as an existing contact or account.
  3. Calendar events more than 3 months into the future are not processed by Nektar until they come within the 3-month window.
  4. The activity might be in the processing pipeline. See Sync latency for expected time durations. To look up what happened to a specific activity, use the Tracker.
Do Google Calendar and MS365 Calendar event updates sync to Salesforce?

Yes, Nektar treats Google/MS365 calendars as the main source of truth. Any changes to the meeting title, date, time, participants, or description will sync back to Salesforce. However, if an internal user has made updates to the event in Salesforce, Nektar will not override those changes. See Self healing for more details.

How does Nektar handle activities that occurred before the account was created in Salesforce?

When the account is created and gets a domain name (from the "website" field or a contact), all people with matching email addresses and their conversations will be created on Salesforce. This is part of Nektar's self-healing feature.

The activity creation date has changed since it was originally created. Why?

Salesforce does not reliably handle certain updates to activities, such as re-associating from a contact to a lead. In rare situations, Nektar has to delete and recreate the activity because of new information it becomes aware of. See Self healing for more details.

Why is Nektar creating old activities now?

Nektar can be configured with a start date, and only activities after that date will be captured. However, Nektar captures activities when it becomes convinced of their relevance to your sales process, which may not be when they actually happened. For example, an old activity may be skipped initially and captured later when a Salesforce account is created with a matching domain name.

Does Nektar create duplicate activities in Salesforce?

Nektar does not create duplicate activities. However, Nektar does not de-duplicate against other activity capture systems running on the same users. To ensure no activities are duplicated, disable other activity capture tools for Nektar users.

An activity has both leads and contacts as participants. Who will Nektar create this activity with?

Nektar prioritizes contacts over leads. Nektar will link an activity with a lead only if there are neither existing contacts nor activity participants that can be created as contacts.

An activity has participants from multiple domains. Where will Nektar create this activity?

Nektar creates this activity with the best-suited Account or Opportunity, using proprietary affinity models. See Graph inference for more information.

How long does it take for Nektar to write activities in Salesforce?

Nektar checks Google or MS365 every 10 minutes and connects with Salesforce APIs every 20 minutes, writing up to 200 records per API call. See Sync latency for more information.

Why do I see duplicate records when I pull activity reports on Salesforce?

This happens when Shared Activities is turned on in your Salesforce. When internal Salesforce users are added as participants, Salesforce creates a copy of the event for each internal user. To avoid inflated counts, all activity reports should set "Event invitations" to false, ensuring activities are not counted multiple times.

Why is an activity linked to an account even when there are open opportunities under that account?

Nektar's affinity models find the best opportunity or account for each activity. When there are multiple opportunities and there is ambiguity about which one the activity relates to, Nektar adds it to the account instead. If this is happening frequently, review your opportunity and opportunity-contact role creation policies to ensure each conversation has an unambiguous opportunity.

Why are activities assigned to users whose inboxes and calendars Nektar is not reading?

Nektar can only access the inboxes and calendars of Nektar users. However, it may assign emails or events to other Salesforce users if the algorithm determines they are a more appropriate owner. See the Owner documentation for more information.

Contacts

With which account and opportunities would a contact be linked?

Nektar creates contacts under the account with a domain name that matches the contact's email address. An account's domain name may come from its "Website" field, or be inferred from the email addresses of other contacts already in it. Nektar considers a person related to an opportunity if they have participated in an activity related to that opportunity. By default, all buyers related to an opportunity are linked to it using OCRs.

Does Nektar create duplicate contacts on Salesforce?

Nektar will not create a contact when it knows of an existing contact or lead with the same email address. However, it may create a duplicate if it is unaware of the existing record — for example if the existing contact was created within the last 20 minutes, or if the integration user's permissions prevent Nektar from seeing it. As soon as Nektar becomes aware of a duplicate, it will resolve it by deleting the contact it created and transferring all activities and fields to the surviving record.

How does Nektar handle contacts whose conversations occurred before the account was created on Salesforce?

With Nektar's self-healing feature, once you have created the account and added a website, all people matching that domain will be created as contacts and conversations with them linked to the account.

I created some contacts on Salesforce. Will Nektar update details like title, phone number, etc.?

Yes, Nektar updates contacts regardless of whether they were created by Nektar or by users.

There are multiple internal people marked on an activity. How does Nektar decide who should be the contact owner?

Contacts are associated with accounts. The owner of the Account with which the Contact was associated is assigned as the Contact owner. See the Owner documentation for more information.

Admin controls

How can I add or remove users?

See the Users documentation.

How can I stop Nektar capturing private calendar meetings?

You can turn off private calendar meetings sync from the blocking section under read rules.

What controls are available to prevent Nektar capturing sensitive information?

Nektar has a robust set of read rules that you can use to define what Nektar should not read. See Data controls for details.

Can I enable read and write of specific Salesforce objects, e.g. contacts without activities?

Yes, Nektar offers the flexibility to enable read/write at an object level.

Can I see a log of all the activities and persons that Nektar read but did not create in Salesforce?

No. The majority of activity in your sellers' inboxes and calendars is not external sales-related communications. As the volume of such activities tends to be excessive and may contain sensitive or personal conversations, we do not provide an interface to review them manually. If you have a specific activity to track down, use the Tracker.

Security

How does Nektar secure my data?

Nektar is compliant with SOC 2 Type II, ISO 27001, and GDPR. See Security for details.

What permissions are required by Nektar?

See the connector documentation for Salesforce, Google Workspace, Microsoft 365, and Zoom.

Why does Nektar need delete permissions on Salesforce?

Nektar's self-healing system re-evaluates past data capture decisions in light of new information, and where necessary reverts changes it made. For example, Nektar might add a contact to an opportunity, but later decide a different opportunity was a better fit — in which case it deletes the first opportunity-contact role and creates a new one. At no point does Nektar delete records it did not originally create, or records that were updated manually after creation.

What custom fields are created by Nektar in Salesforce?

See Nektar custom fields.

How do I know what data Nektar has created in Salesforce?

Review the Dashboard for overall statistics. To review specific contacts, activities and opportunity-contact roles, create a Salesforce report filtering on the NektarActions__c field.

Which IP addresses does Nektar make API calls from?

Nektar makes API calls from one of the following IP addresses: 44.207.41.98, 100.24.77.218, 44.206.28.70, 54.82.117.103.

How many Salesforce APIs does Nektar consume every day?

Nektar is very efficient in its Salesforce API consumption, and on an ongoing basis it consumes under 2,000 API calls (total) per day. For comparison, Salesforce's API quota is 100,000 + (number of licenses × calls per license type) + purchased API Call Add-Ons.

How many objects does Nektar write per API call?

Nektar writes data to Salesforce in batches of 200 objects at a time and continues until all objects have been processed. On average, writing 200 objects takes about 10–30 seconds.

How does Nektar's AI agent, Daisy, use and store customer data?

Your data is processed through OpenAI, Google Gemini, and AWS Bedrock APIs — all SOC 2 compliant. Enterprise agreements explicitly prohibit data storage or use for model training.
